# Security Policy

## Supported Version

| Version | Supported |
|---|---|
| 1.x | Yes |

## Reporting A Vulnerability

If you discover a security issue, please report it privately to the repository owner.

Recommended report contents:
- Affected version and hardware context.
- Reproduction steps.
- Impact analysis.
- Suggested mitigation, if available.

Please do not disclose security-sensitive details publicly before a fix is available.

## Security Considerations For Deployment

- Use strong setup AP credentials before field deployment.
- Restrict physical access to control cabinet and ESP32 board.
- Place devices behind trusted local networks.
- Apply OTA updates only from trusted firmware sources.
